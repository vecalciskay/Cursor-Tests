# Development Plan — Step 02: Bar Chart from JSON Results

## 1. Objective

Implement a Python script that reads aggregated sales data from `outputs/results.json` (produced by Step 01) and generates a bar chart saved as `outputs/barchart.png`.

This is the second step in the broader Excel results reader project described in the README. Step 01 handles Excel ingestion and JSON aggregation; Step 02 visualizes those results. A later step will add an HTML web page.

## 2. Requirements Summary

Source: [step02_chart.md](./step02_chart.md)

| Requirement | Detail |
|-------------|--------|
| Input file | `outputs/results.json` |
| Output file | `outputs/barchart.png` |

| Prerequisite | Step 01 must have been run so `results.json` exists |
| Logging | Standard-library logging at every pipeline step (see Section 7) |

## 3. Input Data Specification

### 3.1 Source

Input is the JSON file written by [scripts/read_excel.py](../../scripts/read_excel.py) in Step 01.

### 3.2 JSON schema

```json
{
  "perSale": [
    { "salesRepName": "Ashish", "totalValue": 6386039.2274457365 },
    { "salesRepName": "Jane", "totalValue": 6665269.382985181 },
    { "salesRepName": "John", "totalValue": 6148152.828352523 }
  ]
}
```

| Field | Type | Usage in chart |
|-------|------|----------------|
| `perSale` | array | Required top-level key; one bar per entry |
| `salesRepName` | string | Category label (x-axis) |
| `totalValue` | number | Bar height (y-axis) |

### 3.3 Known data characteristics (from current `results.json`)

- Three sales representatives: `Ashish`, `Jane`, `John`
- Entries are sorted alphabetically by `salesRepName` (Step 01 convention)
- `totalValue` values are large floats (~6M range)

The chart script should not assume a fixed number of reps; it should render whatever valid entries appear in `perSale`.

## 4. Output Data Specification

### 4.1 PNG bar chart

| Property | Detail |
|----------|--------|
| File path | `outputs/barchart.png` |
| Format | PNG raster image |
| Chart type | Vertical bar chart |
| X-axis | Sales representative names (`salesRepName`) |
| Y-axis | Total sales value (`totalValue`) |
| Bars | One bar per `perSale` entry |

### 4.2 Chart content (recommended defaults)

The requirements do not specify titles, colours, or formatting. Use sensible defaults and document them in the script:

- **Title:** e.g. `Total Sales by Sales Representative`
- **X-axis label:** e.g. `Sales Representative`
- **Y-axis label:** e.g. `Total Sales Value`
- **Value labels:** optional bar value annotations or y-axis tick formatting for readability at ~6M scale
- **Figure size:** e.g. 10×6 inches at 100 DPI — large enough to read rep names and scale
- **Bar order:** preserve order from `perSale` array (alphabetical given Step 01 output)

## 6. Technical Approach

### 6.1 Library choice

Use **matplotlib** to build and save the bar chart. It is the standard choice for static PNG charts in Python and fits this step without adding a web stack.

Add to `requirements.txt`:

```
matplotlib==3.9.4
```

(Pin a recent stable version compatible with the project's Python version.)

### 6.2 Processing flow

Each step below corresponds to an INFO log line (or DEBUG for detail). See Section 7 for the full log catalogue.

```
outputs/results.json
       │
       ▼  [log] Starting chart pipeline; log input/output paths
  Validate file exists and JSON schema
       │
       ▼  [log] Input file found; JSON loaded; perSale entry count
  Extract salesRepName and totalValue from perSale
       │
       ▼  [log] Chart data extracted; rep names and values listed
  Build bar chart (matplotlib)
       │
       ▼  [log] Figure created; chart options applied
  Ensure outputs/ exists → save outputs/barchart.png
       │
       ▼  [log] PNG written; file size; pipeline finished successfully
```

### 6.3 Chart generation logic

1. Load and parse `outputs/results.json`.
2. Validate top-level key `perSale` exists and is a non-empty list.
3. For each item, validate `salesRepName` (non-empty string) and `totalValue` (numeric).
4. Build parallel lists of names and values for plotting.
5. Create figure and axes; draw vertical bars.
6. Apply title, axis labels, and tick formatting as needed.
7. Save to `outputs/barchart.png` with `fig.savefig(..., bbox_inches="tight")`.
8. Close the figure to free memory.

Edge cases to handle:

- Missing `outputs/results.json` — fail with a clear message suggesting Step 01 be run first.
- Missing or empty `perSale` — fail with a clear error.
- Invalid entry (missing keys, blank name, non-numeric value) — fail with index or rep name in the message.
- Very large or very small datasets — matplotlib handles variable bar counts; no special casing required for the current three reps.

## 7. Logging Strategy

Step 02 must use Python’s standard **`logging`** module so every stage of the pipeline is visible when the script runs. Do not rely on bare `print()` for operational messages; reserve prints for nothing (logging only).

### 7.1 Goals

- Trace the full lifecycle: start → load → validate → extract → plot → save → finish.
- Make failures diagnosable: log the failing step, path, and reason before exiting.
- Keep output readable in normal runs (INFO) while allowing deeper detail (DEBUG) when needed.

### 7.2 Configuration

Implement `configure_logging()` called once at script startup (before any pipeline work):

| Setting | Value |
|---------|--------|
| Logger name | `__name__` (module logger for `chart_results.py`) |
| Default level | `INFO` |
| Handler | `logging.StreamHandler(sys.stderr)` — logs go to stderr so stdout stays clean |
| Format | `%(asctime)s %(levelname)s %(name)s: %(message)s` |
| Date format | `%Y-%m-%d %H:%M:%S` |

Optional: honour environment variable `LOG_LEVEL` (e.g. `DEBUG`, `WARNING`) to override the default without code changes. Document this in the root README Step 02 section.

Do **not** add a log file in Step 02; console logging is sufficient. File logging may be considered in a later step if needed.

### 7.3 Log levels by situation

| Level | When to use |
|-------|-------------|
| `DEBUG` | Parsed JSON keys, raw `perSale` item details, matplotlib backend, figure DPI, tick formatter choice |
| `INFO` | Pipeline milestones — each major step in Section 7.4 |
| `WARNING` | Recoverable oddities (e.g. overwriting an existing `barchart.png`) |
| `ERROR` | Validation failures, missing files, save failures — log then exit with code 1 |
| `EXCEPTION` | Use `logger.exception(...)` inside `except` blocks to include stack traces at ERROR level |

### 7.4 Step-by-step log catalogue

Every function in the pipeline should emit at least one INFO message on entry or success. Suggested messages:

| Step | Function / location | Level | Example message |
|------|---------------------|-------|-----------------|
| 1 | `main()` start | INFO | `Starting Step 02 chart generation` |
| 2 | `main()` | INFO | `Input: %s`, `Output: %s` (resolved absolute paths) |
| 3 | `load_results()` start | INFO | `Loading results from %s` |
| 4 | `load_results()` | DEBUG | `Input file size: %d bytes` |
| 5 | `load_results()` success | INFO | `Loaded JSON with %d perSale entries` |
| 6 | `load_results()` / validator | DEBUG | `Validated perSale[%d]: salesRepName=%r totalValue=%s` (per entry) |
| 7 | `extract_chart_data()` | INFO | `Extracted chart data for %d sales representatives` |
| 8 | `extract_chart_data()` | INFO | `Rep %r: totalValue=%s` (one line per rep at INFO, or DEBUG if verbose) |
| 9 | `create_bar_chart()` start | INFO | `Creating bar chart (%d bars)` |
| 10 | `create_bar_chart()` | DEBUG | `Figure size: %sx%s inches, DPI: %s` |
| 11 | `create_bar_chart()` | DEBUG | `Chart title: %r, x-label: %r, y-label: %r` |
| 12 | `create_bar_chart()` save | INFO | `Saving chart to %s` |
| 13 | `create_bar_chart()` success | INFO | `Wrote PNG (%d bytes) to %s` |
| 14 | `main()` success | INFO | `Step 02 completed successfully` |
| 15 | Any failure | ERROR | `Failed to load results: %s` / `Invalid perSale entry at index %d: %s` / etc. |

On failure, log at ERROR (with `logger.exception` when catching unexpected errors), then raise or exit with status code 1 — same behaviour as Step 01, but with structured logs instead of only a single stderr print.

### 7.7 Implementation notes

- Call `configure_logging()` as the first statement in `main()`.
- Pass a `logger` into helpers or use a module-level `logger = logging.getLogger(__name__)`.
- Suppress noisy third-party loggers if needed: `logging.getLogger("matplotlib").setLevel(logging.WARNING)`.
- Do not log entire file contents at INFO; use DEBUG sparingly for per-row/per-entry detail.
